Introduction

The inspiration for this article is twofold: for my love of powerful GIS libraries and my hatred of frustrating installs. Geopandas intersects[1] both.

If you found this article, that means you most likely encountered issues installing the geopandas python library. If not, no worries! If you do not understand the above snippet in green, have not fear. The method below is the smoothest entry (in my experience) to begin coding with one of the popular geospatial python libraries to date.

History

While I won't dive into the entire backstory of Open-source GIS, (though that may be an intriguing article for another time...) I want to provide some context into why we must take this seemingly roundabout approach to simply install a python library.

Geopandas can arguably be described as the culmination of technology from the past two decades of advancement in GIS, python, and software development (1963 if you would like to get technical[3]). I can confidently support this argument due to the fact that geopandas stands on the shoulders of giants; this python library is built on top of established and powerful geospatial code. Although this methodology is practical in order to avoid re-inventing the wheel, this is not without adverse symptoms.

The Problem

From the screenshot above, four out of the nine google search suggestions alude to the core of the issue: simply installing! "But", you say, "I normally use pip, what's so hard about downloading a python library?" Enter dependencies. You know how geopandas stands on the shoulders of giants? Well, you gotta install those giants too.

List of dependencies for geopandas[2]
  • numpy
  • pandas (version 1.0 or later)
  • shapely (interface to GEOS; version 1.7 or later)
  • fiona (interface to GDAL; version 1.8 or later)
  • pyproj (interface to PROJ; version 2.6.1 or later)
  • packaging

The biggest hurdle for installation include those odd little interfaces. Those python packages are ports of C/C++ libraries. And some of those C/C++ libraries are Java ports themselves!

The python community, specifically the scientific computing community notoriously struggled issues with package installation, dependicy errors, and library management. So much so that they created their own Python distribution: Anaconda.
